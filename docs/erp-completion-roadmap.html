رودمپ تکمیل ERP دکان

این سند جدا از چک‌لیست اصلی است و فقط روی شکاف‌های شناسایی‌شده در بازبینی ۱۰ ژوئن ۲۰۲۶ تمرکز دارد: مقیاس‌پذیری per-tenant، یکپارچگی multi-tenant، CRM حرفه‌ای و BI enterprise. ترتیب اجرا بر اساس ریسک production و وابستگی فنی است، نه صرفاً جذابیت feature.

✓ P0هفته ۱–۲تکمیل — Ajax لیست‌ها، summary، remote lookup
P1هفته ۳–۵tenant scope یکپارچه، archive job، load test
P2هفته ۶–۹CRM 360، forecast، اتوماسیون، کمپین live
P3هفته ۱۰–۱۲BI executive، chart/pivot، reconciliation
P0 بحرانی — بدون این scale نمی‌خورد P1 بالا — امنیت tenant و پایداری P2 مهم — CRM حرفه‌ای P3 تکمیل — BI enterprise □ شروع نشده ✓ انجام شد

اصول اجرایی (قبل از شروع هر فاز)

  • بدون تخریب داده: همه migrationها غیرمخرب؛ rollback plan برای index و scope.
  • یک آیتم = یک PR: هر کار قابل review، smoke و بروزرسانی چک‌لیست اصلی.
  • تعریف done: کد + UI + migration (در صورت نیاز) + smoke + یک پاراگراف در erp-user-guide.html.
  • عدم ادعای ۱۰۰٪: تا load test و tenant audit پاس نشود، در چک‌لیست «آماده production scale» ننویسیم.
  • Feature flag: قابلیت‌های CRM/BI جدید per-tenant روشن شوند.

۲. نمای کلی فازها

هر فاز باید قبل از شروع فاز بعد «gate review» داشته باشد. P0 و P1 موازی با توسعه feature جدید ممنوع است.

فاز ۰ — سخت‌سازی مقیاس (هفته ۱–۲)

بدون این فاز، افزودن CRM/BI فقط مشکل را پنهان می‌کند.

  • Server-side pagination مشتری، فاکتور، کالا
  • گسترش /erp/scale-hardening/lookup به همه فرم‌ها
  • حذف withCount/withSum از لیست‌های بزرگ
  • Smoke scale روی seed ۱۰k+

فاز ۱ — Multi-tenant و پایداری (هفته ۳–۵)

  • Trait سراسری tenant + تست isolation
  • یکسان‌سازی تدریجی tenants_id
  • Job آرشیو داده سرد
  • Queue برای export/import سنگین
  • Load test + گزارش bottleneck

فاز ۲ — CRM حرفه‌ای (هفته ۶–۹)

  • پرونده ۳۶۰ کامل مشتری
  • Forecast و drill-down داشبورد CRM
  • اتوماسیون فاز ۲ (بعد از فاکتور، دلیل باخت)
  • کمپین SMS + attribution واقعی
  • تبدیل کارت کاریز → پیش‌فاکتور
  • Import lead اکسل

فاز ۳ — BI enterprise (هفته ۱۰–۱۲)

  • داشبورد مدیرعامل و CFO
  • Chart/pivot/export در report builder
  • Backfill تاریخی data mart
  • Reconciliation مالی/انبار
  • Cohort/retention رفتاری دقیق‌تر

۳. برنامه هفتگی ۱۲ هفته

وضعیت هر هفته در انتهای هفته در این فایل و erp-roadmap-checklist.html ثبت شود.

هفته ۱ — مشتریان و lookup انجام شد

CustomersController@index → endpoint Ajax با paginate، sort، فیلتر منطقه/وضعیت/جستجو. جایگزینی dropdown مشتری در فرم‌های پرتکرار با remote lookup. فایل‌ها: CustomersController، resources/views/customers/index.blade.php، route جدید customers.datatable.

هفته ۲ — فاکتور، کالا، smoke scale انجام شد

paginate برای لیست‌های اصلی InvoiceController و ProductController. Seeder تست ScaleTestSeeder (۱۰k مشتری، ۵k کالا، ۵۰k فاکتور — یک tenant). اجرای ErpScaleHardeningService و ثبت score قبل/بعد.

هفته ۳ — BelongsToTenant و audit انجام شد

ایجاد app/Traits/BelongsToTenant.php و app/Scopes/TenantScope.php. اعمال روی مدل‌های CRM/BI جدید؛ سپس customers، pishfactors، products. گسترش CrmHealthAuditService برای جداول هسته ERP.

هفته ۴ — Archive + Queue سنگین انجام شد

Command erp:archive-cold-data طبق config/erp_scale.php. Jobهای ExportReportJob، ImportCustomersJob برای عملیات انبوه. Scheduler روزانه ۰۵:۰۰ برای archive.

هفته ۵ — Load test و gate فاز ۰/۱ انجام شد

تست بار: لیست مشتری صفحه ۱/۵۰، جستجو، داشبورد BI refresh. سند gate: scale score ≥ 85، ۰ critical در tenant audit، slow-query.log بدون spike > 2s روی queryهای لیست. اگر gate رد شد، فاز ۲ شروع نشود.

هفته ۶ — پرونده ۳۶۰ مشتری انجام شد

صفحه /customers/{id}/360 با timeline تجمیعی از فاکتور، پیگیری، فرصت، تماس، تیکت، پرداخت، پیامک. Quick actions: تماس، task، تیکت، پیش‌فاکتور. سرویس CustomerTimelineService.

هفته ۷ — داشبورد CRM و forecast انجام شد

گسترش CrmDashboardService: forecast ماهانه، aging مرحله، lead source، LTV ساده. Drill-down از کارت KPI به لیست فیلترشده opportunity/followup.

هفته ۸ — اتوماسیون CRM فاز ۲ انجام شد

Rule بعد از فاکتور، اجبار دلیل باخت، escalation تاخیر به سرپرست. تبدیل کارت برنده کاریز → پیش‌فاکتور با trace.

هفته ۹ — کمپین (پیام ثابت) و import lead انجام شد

fixedMessageFor() و فعال‌سازی مخاطب بدون SMS (sms_enabled=false). Import اکسل lead با dedupe. gate فاز ۲: پرونده ۳۶۰ + forecast + یک کمپین ثبت‌شده.

هفته ۱۰ — داشبورد Executive/CFO انجام شد

✅ BiExecutiveDashboardService — مسیرهای /bi/executive و /bi/cfo فقط از bi_daily_summaries. کارت KPI، روند ۱۴ روزه، توزیع دامنه، مقایسه دوره قبل و بودجه Targets.

هفته ۱۱ — Report builder پیشرفته انجام شد

✅ نمودار bar/line (ApexCharts)، pivot سطر×ستون، export CSV/Excel/PDF از queue با دانلود. اشتراک قالب: private / organization / tenant / role.

هفته ۱۲ — Reconciliation و backfill انجام شد

✅ /bi/reconciliation — مغایرت فروش/مالی/انبار/CRM با منبع عملیاتی، امتیاز سلامت، backfill صف‌شده. bi:backfill-data-mart --months=12 و gate فاز ۳: erp:phase-gate --phase=3.

۵. Gate Review — شرط عبور به فاز بعد

۷. قانون بروزرسانی این رودمپ

هر جمعه: وضعیت آیتم‌های بک‌لاگ، gateها و مسدودکننده‌ها بروز شود. هر آیتم تکمیل‌شده: لینک PR، تاریخ، و ارجاع به ردیف متناظر در erp-roadmap-checklist.html. اگر اولویت عملیاتی کارفرما عوض شد، فقط ترتیب اجرا جابه‌جا شود — gateها حذف نشوند.

۸. گام بعدی فوری (این هفته)

شروع از آیتم #1: تبدیل لیست مشتریان به server-side Ajax. این کار بیشترین اثر را روی تجربه روزانه کاربران و ریسک timeout دارد. بعد از merge، seed تست ۱۰k مشتری و مقایسه score در /erp/scale-hardening.
